# 코드 스타일

- 함수는 짧게 — **한 가지 일만**, 20줄 이내 권장.
- 네이밍은 일관되고 의미 있게 — {{camelCase}} (프로젝트 컨벤션에 맞게 교체).
- 주석은 **"왜"**를 적는다("무엇"은 코드로 드러낸다). 주석 언어: {{영어}}.
- 조기 추상화 금지 — 같은 패턴이 3번 나오기 전엔 일반화하지 않는다.
- 기존 코드의 스타일을 따른다(주변 코드처럼 읽히게). 새 스타일을 끼워넣지 않는다.
