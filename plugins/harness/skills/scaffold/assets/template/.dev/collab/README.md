# .dev/collab/ — 에이전트 간 협업 (작업 인계 · 동시 작업 조율)

여러 에이전트(또는 세션)가 한 프로젝트를 **이어서 / 동시에** 작업할 때,
서로 *무엇을 했고 · 무엇을 잡았고 · 무엇을 넘기는지*를 파일로 공유하는 곳.
사람 진실인 `docs/`, 개인 학습인 `.dev/learnings/`와 섞지 않는다 — 여기는 **조율 상태**만 둔다.

> 단일 에이전트로만 일한다면 이 폴더는 필요 없다. 여러 에이전트/세션이 협업할 때만 둔다.
> (좋은 Harness는 점점 단순해진다 — 안 쓰면 지운다.)

## 무엇이 있나

| 파일/폴더 | 용도 | git |
|-----------|------|-----|
| `handoff/<topic>.md` | 인수인계서 — 작업을 넘길 때 남기는 기록 (다음 에이전트가 가장 먼저 읽음) | **커밋** (공유·리뷰) |
| `handoff/TEMPLATE.md` | 인수인계서 양식 | 커밋 |
| `DECISIONS.md` | 에이전트 간 가벼운 결정 로그 | **커밋** |
| `BOARD.md` | 작업 보드 — 지금 누가 무엇을(진행/대기/완료) | **추적 안 함** (라이브·휘발성 · 워크트리 간 심링크 공유) |
| `claims/<task>.md` | 작업 선점 락 — 동시 작업 충돌 방지 | **추적 안 함** (휘발성 · 워크트리 간 심링크 공유) |

> **분리 원칙**: 오래 남길 기록(인계서·결정)은 **커밋**(git으로 동기화), 휘발성 라이브 상태(보드·락)는 **gitignore**.
> 보드/락은 git에 올리면 머지 충돌만 나므로 추적하지 않는다. 대신 워크트리 간에는 **심볼릭 링크**로 한 정본을 공유한다(아래 참고).
> gitignore 라인은 프로젝트 루트 `.gitignore`에 있다 (`/.dev/collab/BOARD.md`, `/.dev/collab/claims/`).

## 프로토콜 (에이전트가 따르는 순서)

1. **시작 전** — `handoff/`(이어받을 작업) + `BOARD.md`(진행 중)를 먼저 읽는다. 같은 작업을 중복으로 잡지 않는다.
2. **선점** — 잡을 작업은 `claims/<task>.md`를 만들어 표시(누가·언제·범위) 후 `BOARD.md`에 한 줄 추가한다.
   범위가 겹치면 **먼저 잡은 쪽 우선**, 조율한다.
3. **작업 중** — 공유에 영향 주는 결정(포맷·인터페이스·되돌리기 어려운 선택)은 `DECISIONS.md`에 한 줄 남긴다.
4. **넘길 때·끝낼 때** — `handoff/<topic>.md`에 인수인계서를 쓴다(끝낸 것 / 남은 것 / 함정 / 다음 단계 / 관련 파일·커밋).
   `BOARD.md` 상태를 갱신하고, `claims/`의 락은 해제(파일 삭제)한다.
5. **다음 에이전트** — `handoff/`를 가장 먼저 읽고 이어간다.

## 워크트리 기반 협업 + 공유 파일 심볼릭 링크 (동시 작업의 기본형)

여러 에이전트가 **동시에** 일할 때는 각 에이전트를 **git worktree**로 격리한다 — 작업 트리·인덱스 충돌을 원천 차단한다.

```bash
# primary 체크아웃(정본)에서, 에이전트마다 워크트리 1개씩
git worktree add ../<project>-agentA -b agent/A
git worktree add ../<project>-agentB -b agent/B
```

**핵심 함정:** worktree는 **커밋 히스토리(추적 파일)는 공유**하지만 **ignore/untracked 파일은 워크트리마다 따로** 가진다.
즉 gitignore된 `BOARD.md`·`claims/`는 워크트리마다 **갈라져** 서로의 선점을 못 본다 → 조율이 깨진다.

**규칙 — 공유 파일은 심볼릭 링크로 단일 정본을 가리킨다:**
정본은 **primary 체크아웃**의 `.dev/collab/`. 각 에이전트 워크트리에서 휘발성 공유 파일을 그 정본으로 링크한다.

```bash
# 각 에이전트 워크트리 안에서 (PRIMARY=정본 체크아웃의 절대경로)
PRIMARY=/abs/path/to/<project>            # primary 체크아웃 루트
ln -sfn "$PRIMARY/.dev/collab/BOARD.md"  .dev/collab/BOARD.md
ln -sfn "$PRIMARY/.dev/collab/claims"    .dev/collab/claims
```

- **링크 대상 = gitignore된 휘발성 파일뿐** (`BOARD.md`, `claims/`). 이것들이 곧 "git이 못 나르는" 파일이라 링크가 필요하다.
- **추적 파일(`handoff/`·`DECISIONS.md`)은 링크하지 않는다.** git으로 동기화된다 — 넘길 때 커밋하고, 공유 브랜치(main 등)로 머지/리베이스해 다른 워크트리가 받는다.
- 링크 자체는 gitignore 경로와 같아 **추적되지 않는다**(추가 .gitignore 불필요). 경로 이식성보다 단순함을 택해 **절대경로**로 건다(휘발성이라 커밋 안 됨).
- 선점할 때 `claims/<task>.md`에 **어느 워크트리/브랜치**가 잡았는지 적는다(예: `agent/A · ../<project>-agentA`).

> 같은 머신·단일 작업 트리에서 순차로만 일한다면 워크트리·링크는 필요 없다. 동시 격리가 필요할 때만 쓴다(원칙: 안 쓰면 만들지 않는다).

## 개선 루프 (쌓이면 승격, 끝나면 정리)

- 같은 함정이 인계서에 반복되면 → `.dev/learnings/` 또는 `.claude/rules/`로 승격.
- 결정이 굳어 아키텍처 수준이면 → `docs/adr/`(정식 ADR)로 올리고 여기선 지운다.
- 끝난 작업의 `claims/` 락과 `BOARD.md` 줄은 **바로 치운다** — 쌓이면 노이즈이고 조율이 거짓말을 한다.
